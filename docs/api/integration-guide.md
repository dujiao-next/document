# 站点对接说明

> 更新时间：2026-03-01

本文用于说明 Dujiao-Next 站点之间如何进行对接与串货，帮助站长快速跑通从“开通权限”到“下单采购”的完整流程。

## 1. 先理解两个角色

- 对接方（A 站）：抓取上游商品、在自己站点售卖，并在用户下单后向上游发起采购。
- 被对接方（B 站）：开放接口给其他站点调用，接收采购请求并从对接用户钱包扣款。

## 2. 标准流程（A 对接 B）

1. B 站用户在前台开通 API 权限，生成 Access Key / Secret。
2. A 站管理员在后台新增“对接连接”，填写 B 站地址和凭证。
3. A 站测试连接成功后，拉取 B 站商品并建立商品映射。
4. A 站用户下单后，A 站会向 B 站发起采购单。
5. B 站按规则处理订单并扣减该对接用户的钱包余额。

## 3. 被对接方（B 站）需要做什么

### 3.1 设置开放接口地址

后台路径：`系统设置 -> 品牌与站点`，设置“开放接口地址（Open API URL）”，例如：

- `https://api.example.com/api/open/v1`

这个地址会展示给前台用户，用于给对接方提供调用入口。

### 3.2 用户开通 API 权限并生成 Token

前台路径：`个人中心 -> API 权限`

用户可执行以下操作：

- 开通 API 权限
- 创建多个 Token（Access Key / Secret）
- 设置 Token 过期时间
- 启用/停用或删除 Token

注意：`Secret` 只在创建时展示一次，请立即保存。

## 4. 对接方（A 站）如何新增连接

后台路径：`站点对接 -> 对接连接 -> 新增`

建议填写规则：

| 字段 | 填写说明 |
| --- | --- |
| 连接名称 | 自定义，便于区分来源站点 |
| 站点地址 | 填 B 站根地址，如 `https://b.example.com` |
| 协议类型 | 使用默认值（Dujiao OpenAPI v1） |
| 协议版本 | 使用默认 `v1` |
| Access Key | B 站用户生成的 Access Key |
| Secret | B 站用户生成的 Secret |
| 回调地址 | 可选，建议填写 A 站可公网访问地址 |

注意：

- “站点地址”建议填写根地址，不要手动拼 `/api/open/v1`，系统会按协议固定路径调用。

## 5. 拉取商品与建立映射

后台路径：`站点对接 -> 商品映射`

建议流程：

1. 先选择连接并拉取上游商品。
2. 选择本地商品与上游商品/SKU 的映射关系。
3. 按你的运营策略设置本地价格与展示信息。
4. 启用映射后再对外售卖。

## 6. 下单与扣款行为

- A 站用户下单后，系统会自动向 B 站创建采购单。
- B 站会从该对接用户的钱包扣款。
- 若余额不足，采购会立即失败并返回错误，不会延迟扣款。
- 若库存不足或商品不可用，也会立即返回失败。

说明：

- 未开启“余额不足自动停售”时，映射商品不会自动下架，但订单会按失败状态返回。
- 前台用户不会看到“第三方对接来源”标识，来源信息仅后台可见。

## 7. 运营排查建议

### A 站（对接方）重点看

- `站点对接 -> 对接订单`：查看采购状态、失败原因、重试情况。
- `站点对接 -> 回调日志`：查看上游状态回传是否成功。

### B 站（被对接方）重点看

- `站点对接 -> 被对接用户`：查看哪些用户已开通 API 权限、Token 使用情况。
- `站点对接 -> 对接订单`：筛选 `source=inbound` 查看外部站点采购单。

## 8. 常见问题

### Q1：测试连接返回 401

优先检查：

- Access Key / Secret 是否填错
- Token 是否已停用或过期
- 站点地址是否可公网访问

### Q2：下单时报参数或签名错误

优先检查：

- 请求是否带签名头
- 写接口是否带 `X-DJ-Idempotency-Key`
- 时间戳是否偏差过大

### Q3：用户下单成功但采购失败

常见原因：

- B 站钱包余额不足
- 上游库存不足
- 商品映射关系错误（商品 ID / SKU ID 不匹配）
